# Электронная сеть продаж электроники

Django-приложение для управления иерархической сетью по продаже электроники 
с API интерфейсом и админ-панелью.

## Основные возможности

- Иерархическая структура из трех уровней: Завод → Розничная сеть → Индивидуальный предприниматель

- REST API с CRUD операциями для всех объектов сети

- Админ-панель с расширенными возможностями управления

- Автоматическое определение уровня иерархии на основе отношений

- Валидация бизнес-правил на уровне моделей и API

- Фильтрация и поиск по различным параметрам

- Права доступа к API (только аутентифицированные, активные и сотрудники имеют доступ к API)

## Архитектура

### Модели данных

**Звенья сети (приложение network, модель NetworkLink)**
- звено сети с иерархическими связями
- CRUD-операции (через ModelViewSet)
- Фильтрация объектов по стране (через `django-filter`)
- Поиск по полям наименованию сети, email, городу, стране
- Сортировка по времени создания и сумме задолженности перед поставщиком

**Продукты (приложение network, модель Product)**

 - продукты связанны со звеньями сети
 - CRUD-операции (через ModelViewSet)

#### Уровни иерархии

- Уровень 0: Завод (не имеет поставщика)

- Уровень 1: Розничная сеть или ИП (поставщик - завод)

- Уровень 2: Розничная сеть или ИП (поставщик - уровень 1)

*Каждое звено сети ссылается только на одного поставщика оборудования (не обязательно предыдущего 
по иерархии). Уровень иерархии определяется не названием звена, а отношением к остальным 
элементам сети, т. е. завод всегда находится на уровне 0, а если индивидуальный предприниматель 
относится напрямую к заводу, минуя остальные звенья (розничную сеть), его уровень — 1.)*

**Пользователи (приложение users, модель CustomUser)**

- Регистрация
- Аутентификация и авторизация JWT 
(через `djangorestframework-simplejwt`)
- Профиль расширен аватаром, номером телефона и городом, авторизация по email

## Основные эндпоинты API

### Аутентификация

#### Метод / Endpoint / - Описание

- POST /api/token/ – получение JWT токена

- POST /api/token/refresh/ – обновление токена

### Пользователи

- POST /api/users/register/ – регистрация

### Звенья сети
#### Метод / Endpoint / - Описание
- GET	/api/network/networks/ - Список всех звеньев сети
- POST	/api/network/networks/ - Создание нового звена
- GET	/api/network/networks/{id}/ - Детали звена
- PUT	/api/network/networks/{id}/ - Обновление звена
- PATCH	/api/network/networks/{id}/ - Частичное обновление
- DELETE /api/network/networks/{id}/ - Удаление звена

### Продукты
#### Метод / Endpoint / - Описание
- GET	/api/network/products/ - Список всех звеньев сети
- POST	/api/network/products/ - Создание нового звена
- GET	/api/network/products/{id}/ - Детали звена
- PUT	/api/network/products/{id}/ - Обновление звена
- PATCH	/api/network/products/{id}/ - Частичное обновление
- DELETE /api/network/products/{id}/ - Удаление звена

## Фильтрация API
- GET /api/network/networks/?country=Russia - фильтрация по стране
- GET /api/network/networks/?search=Москва - поиск по названию, email, городу
- GET /api/network/networks/?ordering=created_at - сортировка по времени создания, по возрастанию (-created_at, по убыванию)
- GET /api/network/networks/?ordering=debt_to_supplier - сортировка по задолженности перед поставщиком

### Админ-панель

- Ссылка на поставщика с каждого объекта

- Фильтр по названию города

- Фильтр по названию страны

- Фильтр по созданию 

- Фильтр по названию сети

- Admin action для очистки задолженности

#### Использование действий

- Выберите действие "Очистить задолженность перед поставщиком"
- Отметьте галочками нужные объекты
- Нажмите "Выполнить"

## Бизнес-правила

### Валидация на уровне модели и API

- Завод не может иметь поставщика

- Розничная сеть и ИП должны иметь поставщика

- Нельзя выбрать себя в качестве поставщика

- Максимальный уровень иерархии - 2 (0, 1, 2)

- Задолженность не может обновляться через API

### Автоматические вычисления
- Уровень иерархии вычисляется автоматически при сохранении
- Время создания звена сети устанавливается автоматически

## Установка и запуск

### 1. Клонируем репозиторий
```
git clone https://github.com/Sermin22/TradingNetworkPlatformProjectDRF.git

```
В настоящее время проект выполнен в качестве тестового задания,
поэтому необходимо использовать рабочую ветку, а соответственно переключиться
в рабочую ветку:
```
git checkout feature/working_branch_1
```

### 2. Создайте файл .env  
В корне проекта создайте файл .env и укажите необходимые переменные окружения. 

Пример файла - .env_template

### 3. Установить зависимости:

```
pip install -r requirements.txt
```
### 4. Создать базу данных.

### 5. Выполнить миграции:

```
python manage.py migrate
```

### 6. Запустить локально
```
python manage.py runserver
```

## После запуска:

- Swagger-документация: http://127.0.0.1:8000/swagger/ или http://localhost:8000/swagger/
- Альтернативная документация: http://127.0.0.1:8000/redoc/ или http://localhost:8000/redoc/
